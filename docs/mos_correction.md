# MOS-коррекция bias ERA5 → станция

## Что сделано

Обнаружен систематический bias между данными ERA5 (на которых обучена модель)
и реальными наблюдениями метеостанций в Красноярске. Реализована MOS-коррекция
(Model Output Statistics) для компенсации этого bias при live-прогнозе.

## Проблема

- Модель обучена на ERA5 → предсказания воспроизводят ERA5, а не станцию
- ERA5 не учитывает городской остров тепла (Urban Heat Island, UHI)
- Разница ERA5 vs станция WMO 29570: **от -6°C до +7°C** в зависимости от
  сезона и времени суток
- На защите жюри увидит прогноз, расходящийся с реальной погодой → нужна коррекция

## Ключевые результаты (bias = станция − ERA5, °C)

```
         0h    3h    6h    9h   12h   15h   18h   21h  (UTC)
M01   +7.1  +7.2  +6.2  +6.3  +7.0  +6.7  +6.2  +6.8   ← зима: станция теплее
M04   +0.5  -3.8  -4.1  -2.5  -0.4  +2.6  +2.5  +1.9   ← весна: днём ERA5 теплее
M07   -2.1  -6.2  -5.2  -3.9  -2.3  +1.4  +2.4  +1.0   ← лето: днём ERA5 сильно теплее
M10   +3.9  +2.0  -0.0  +1.8  +4.9  +5.0  +4.6  +4.1   ← осень: станция теплее
```

- Зима: станция ВСЕГДА теплее ERA5 на 5-7°C (UHI + инверсии)
- Лето днём: ERA5 теплее станции на 3-6°C (микроклимат)
- Лето вечером: станция теплее ERA5 на 1-4°C (классический UHI)
- Март (сейчас): днём ≈0°C, вечером +3-4°C

## Данные

- **Станция**: WMO 29570 "Krasnoyarsk Opytnoe Pole" (56.017°N, 92.750°E)
  - Ближайший узел сетки: (56.00, 92.75), расстояние 1.9 км
  - Источник: NOAA ISD-Lite (бесплатно, быстро)
- **ERA5**: Open-Meteo Archive API (бесплатно)
- **Период**: 2020-01-01 – 2024-12-31, 42688 совпавших часовых записей
- **Таблица**: `bias[month][hour_utc]` — среднее смещение за 5 лет

## Созданные файлы

1. **`scripts/build_mos_table.py`** — скрипт сбора данных и расчёта таблицы
   - ERA5: Open-Meteo Archive API (curl, по месяцам)
   - Станция: NOAA ISD-Lite (gz-файлы, по годам)
   - Выход: JSON с `bias_table[month][hour_utc]`

2. **`src/postprocessing/__init__.py`** + **`src/postprocessing/mos_correction.py`**
   - `load_mos_table(path)` — загрузка JSON
   - `get_t2m_bias(table, valid_time)` — bias для конкретного времени
   - `apply_mos_t2m(prediction_phys, var_order, table, valid_times)` — коррекция

3. **`live_runtime_bundle/mos_bias_t2m.json`** — готовая таблица bias (42688 записей)

4. **`scripts/live_gdas_forecast.py`** — добавлен флаг `--mos-table`
   - После денормализации применяет MOS-коррекцию к t2m
   - Печатает bias для каждого горизонта

## Использование

```bash
# Прогноз с MOS-коррекцией:
python scripts/live_gdas_forecast.py \
  --runtime-bundle live_runtime_bundle \
  --mos-table live_runtime_bundle/mos_bias_t2m.json

# Пересчёт таблицы (если нужно обновить):
python scripts/build_mos_table.py --start-date 2020-01-01 --end-date 2024-12-31
```

## Что даёт MOS

- Live-прогноз t2m будет ближе к реальным станционным наблюдениям
- Зимой: вместо "-20°C (ERA5)" покажет "-14°C" (ближе к станции)
- Март вечером: вместо "+4°C" покажет "+8°C" (ближе к krasecology +6.3°C)
- Для защиты: можно показать "прогноз с MOS" vs "факт со станции" — совпадение будет намного лучше
